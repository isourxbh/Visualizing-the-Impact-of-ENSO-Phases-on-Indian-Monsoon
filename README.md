# Visualizing the Impact of ENSO Phases on Indian Monsoon

A comprehensive, interactive web dashboard exploring the historical impact of the El Niño-Southern Oscillation (ENSO) on the Indian Summer Monsoon. 

This project combines sea surface temperature anomalies, historical precipitation data, and vegetation indices into an accessible, high-performance visualization suite. It features a React/Vite frontend and a FastAPI backend powered by SQLite and Python.

## Project Architecture

- **Frontend:** React 18, Vite, Recharts, TailwindCSS v4, shadcn/ui.
- **Backend:** FastAPI, SQLite, pre-computed JSON, data processing via Python (xarray, pandas, scipy, rasterio).
- **Data Store:** SQLite database, pre-computed JSON, and raw NetCDF/GeoTIFF datasets.

## Key Features

- **SST Dataset Comparison:** Interactive grid heatmap of Sea Surface Temperature (SST) anomalies (Niño-3.4).
- **ONI Timeline Strip:** Historical Oceanic Niño Index (ONI) timeline from 1950 to present, allowing precise brushing to filter dashboard data globally.
- **Monsoon Onset Animation:** Animated choropleth maps simulating the daily progression and accumulation of the Indian summer monsoon.
- **Seasonal Rainfall Anomaly:** State-level maps depicting rainfall deviations from the Long Period Average (LPA).
- **ONI vs. Rainfall Statistics:** Scatter plots and sensitivity heatmaps detailing the correlation between ENSO events and regional rainfall.
- **NDVI vs. ONI Integration:** Dual-axis charts exploring vegetation health (NDVI) alongside ENSO indices over time.
- **Global Compare Mode:** A comprehensive "Year A vs Year B" comparison mode syncing dual maps, heatmaps, and charts across the dashboard.

## Setup Instructions

### Prerequisites
- Node.js (for frontend)
- Python 3.10+ (for backend/data processing)
- uv or pip (for Python dependencies)

### Data Setup

**Note:** The `data/` directory is excluded from source control (via `.gitignore`) because it contains large raw datasets (NetCDF, GeoTIFF) and processed SQLite databases.

Before running the application, ensure that you have populated the `data/` directory with the required files (e.g., ERSST v6, OISST v2.1, CHIRPS Rainfall, MODIS NDVI) or the pre-computed `climate.db` database generated by the backend processing scripts.

#### Data Processing Pipeline

If you are starting from raw data, use the provided Python scripts in the `backend/scripts/` directory to generate the SQLite database and pre-computed tables. You should run these scripts sequentially from within the `backend` directory:

```bash
cd backend
uv run python scripts/00_init_db.py
uv run python scripts/01_parse_ersst.py
uv run python scripts/02_parse_oisst.py
uv run python scripts/03_load_rainfall.py
uv run python scripts/04_compute_oni.py
uv run python scripts/05_compute_rainfall_anomalies.py
uv run python scripts/06_compute_correlations.py
uv run python scripts/07_parse_ndvi.py
```

### Running the Backend

The backend is built with FastAPI. It runs on `http://localhost:8000`.

```bash
cd backend
uv run uvicorn backend.main:app --reload
```
*(If you are not using `uv`, you can run `pip install -r requirements.txt` and then `uvicorn backend.main:app --reload`)*

### Running the Frontend

The frontend is a React SPA built with Vite. It proxies `/api/*` to the local backend.

```bash
cd Frontend
npm install
npm run dev
```

### Access the Dashboard

Once both servers are running, access the dashboard at:
`http://localhost:5173`
